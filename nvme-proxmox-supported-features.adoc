= 了解 Proxmox 主机对ONTAP 的支持和功能
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
使用 NVMe over Fabrics (NVMe-oF) 进行主机配置所支持的功能因ONTAP和 Proxmox 的版本而异。

[cols="30,20,20"]
|===
| ONTAP功能 | Proxmox主机版本 | ONTAP 版本 


| 使用原生功能提供 NVMe/FC 和 NVMe/TCP 命名空间的ONTAP详细信息 `nvme-cli`包裹。  a| 
9.0
| 9.10.1 或更高版本 


| NVMe/TCP 是一项完全受支持的企业级功能  a| 
9.0
| 9.10.1 或更高版本 


| 同一主机上支持 NVMe 和 SCSI 流量，NVMe-oF 命名空间使用 NVMe 多路径，SCSI LUN 使用 dm 多路径。  a| 
9.0
| 9.4 或更高版本 
|===
无论系统设置中运行的ONTAP版本如何， ONTAP支持以下 SAN 主机功能。

[cols="70,30"]
|===
| 功能 | Proxmox主机版本 


| 默认情况下启用原生 NVMe 多路径。  a| 
9.0



| 这 `nvme-cli`该软件包包含自动连接脚本，无需第三方脚本。  a| 
9.0



| 原生 udev 规则 `nvme-cli`该软件包为 NVMe 多路径提供轮询负载均衡  a| 
9.0

|===

NOTE: 有关受支持配置的详细信息，请参阅link:https://mysupport.netapp.com/matrix/["互操作性表工具"^]。
