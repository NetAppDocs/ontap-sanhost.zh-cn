请确保为主机正确配置 iSCSI。

.关于此任务
在 iSCSI 主机上执行以下步骤。

.步骤
. 验证是否已安装 iSCSI 启动程序包 (open-iscsi)：
+
[source, cli]
----
$apt list |grep open-iscsi
----
+
您应该会看到类似于以下示例的输出：

+
[listing]
----
open-iscsi/stable,now 2.1.11-1+deb13u1 amd64
----
. 验证位于  `/etc/iscsi/initiatorname.iscsi` 文件中的 iSCSI 启动程序节点名称：
+
[source, cli]
----
InitiatorName=iqn.YYYY-MM.com.<vendor>:<host_name>
----
. 配置位于  `/etc/iscsi/iscsid.conf` 文件中的 iSCSI 会话超时参数：
+
[source, cli]
----
node.session.timeo.replacement_timeout = 5
----
+
iSCSI `replacement_timeout` 参数控制 iSCSI 层在其上的任何命令失败之前应等待超时路径或会话自行重新建立多长时间。您应在 iSCSI 配置文件中将  `replacement_timeout` 的值设置为 5。

. 启用 iSCSI 服务：
+
[source, cli]
----
$systemctl enable iscsid
----
. 启动 iSCSI 服务：
+
[source, cli]
----
$systemctl start iscsid
----
. 验证 iSCSI 服务是否正在运行：
+
[source, cli]
----
$systemctl status iscsid
----
+

. 发现 iSCSI 目标：
+
[source, cli]
----
$iscsiadm --mode discovery --op update --type sendtargets --portal <target_IP>
----
+

====
. 登录到目标：
+
[source, cli]
----
$iscsiadm --mode node -l all
----
. 将 iSCSI 设置为在主机启动时自动登录：
+
[source, cli]
----
$iscsiadm --mode node -T <target_name> -p <ip:port> -o update -n node.startup -v automatic
----
+
将  `<target_name>` 替换为主机启动时要自动登录的目标名称。将  `<ip:port>` 替换为目标的 IP 地址和端口号。例如：

+
[source, cli]
----
iscsiadm --mode node -T iqn.1992-08.com.netapp:sn.c770be5d934811f0b624d039eac809ba:vs.5 -p 192.168.166.70:3260 -o update -n node.startup -v automatic
----
. 验证 iSCSI 会话：
+
[source, cli]
----
$iscsiadm --mode session
----
+
